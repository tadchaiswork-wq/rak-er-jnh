# 💗 รักและใส่ใจ ER JNH

เว็บแอปประเมินการทำงานเพื่อนร่วมงานของ ER JNH — สแกนบาร์โค้ดเข้าเวร รายงาน และชื่นชมเพื่อนร่วมงาน
รันบน **GitHub Pages** (ฟรี) ใช้ **Firebase** เป็นฐานข้อมูลกลาง (ใช้ร่วมกันหลายคน/ข้ามเครื่อง)

---

## 🧩 สิ่งที่ต้องเตรียม (ทำครั้งเดียว)

1. บัญชี Google (สำหรับ Firebase)
2. บัญชี GitHub (สำหรับ host เว็บ)

---

## ขั้นที่ 1 — สร้างโปรเจกต์ Firebase

1. ไปที่ https://console.firebase.google.com → **Add project** → ตั้งชื่อ เช่น `rak-er-jnh` → สร้าง (ปิด Google Analytics ได้)
2. เมนูซ้าย **Build → Authentication** → **Get started** → แท็บ **Sign-in method** → เปิด **Email/Password** → Save
3. เมนูซ้าย **Build → Firestore Database** → **Create database** → เลือก **Start in production mode** → เลือก location `asia-southeast1` (สิงคโปร์) → Enable
4. แท็บ **Rules** ของ Firestore → ลบของเดิม แล้ววางเนื้อหาจากไฟล์ **`firestore.rules`** ในโปรเจกต์นี้ → **Publish**

## ขั้นที่ 2 — เอาค่า config มาใส่

1. ในหน้า Firebase → ไอคอนเฟือง ⚙️ (Project settings) → เลื่อนลงหัวข้อ **Your apps** → กดไอคอน `</>` (Web)
2. ตั้งชื่อ app → Register app → จะได้โค้ด `firebaseConfig = { ... }`
3. เปิดไฟล์ **`js/firebase-config.js`** ในโปรเจกต์นี้ แล้วแทนค่า `apiKey`, `projectId`, ฯลฯ ให้ตรงกับที่ได้มา

## ขั้นที่ 3 — อัปโหลดขึ้น GitHub & เปิด Pages

1. สร้าง repository ใหม่บน GitHub เช่น `rak-er-jnh`
2. อัปโหลดไฟล์ทั้งหมดในโฟลเดอร์นี้ (index.html, scan.html, manifest.webmanifest, โฟลเดอร์ css, js, **icons**, firestore.rules, README.md)
3. ที่ repo → **Settings → Pages** → Source เลือก **Deploy from a branch** → Branch = `main` / folder = `/ (root)` → Save
4. รอสักครู่ จะได้ลิงก์ เช่น `https://ชื่อคุณ.github.io/rak-er-jnh/`

> ⚠️ การสแกนกล้องต้องใช้ **HTTPS** — GitHub Pages เป็น HTTPS อยู่แล้ว ใช้ได้เลย

## ขั้นที่ 4 — สร้างบัญชีแอดมินใหญ่ (ครั้งแรก)

1. เปิดเว็บ → กด **สมัครสมาชิก**
2. กรอกชื่อ–สกุลของคุณ, username = **`pettoo`**, รหัสผ่าน = **`pett1809`**
3. ระบบจะตั้งให้เป็น **แอดมินใหญ่** อัตโนมัติ (เพราะ username = pettoo)

เสร็จแล้ว! คนอื่นเข้าเว็บเดียวกัน กด "สมัครสมาชิก" กรอกชื่อ–สกุล จะได้บัญชี **สมาชิก** พร้อมบาร์โค้ดของตัวเอง

---

## 🖥️ ตั้งเครื่องสแกนเข้าเวรที่ ER (สำคัญ)

การสแกนเข้าเวรทำได้ที่ **เครื่องสแกนหน้างาน ER เท่านั้น** (คอม/iPad ที่ ER) พนักงานกดเข้าเวรเองจากมือถือส่วนตัวไม่ได้ — ต้องเอาบาร์โค้ดของตัวเองมาให้เครื่องที่ ER สแกน

วิธีตั้ง (ทำครั้งเดียว):
1. เข้าเว็บด้วยบัญชี **แอดมินใหญ่ (pettoo)** → แท็บ **จัดการผู้ใช้** → **+ เพิ่มผู้ใช้**
2. สร้างบัญชีสำหรับเครื่องสแกน เช่น username `erscan`, ตั้งรหัสผ่าน, เลือกสิทธิ์ **เครื่องสแกน**
3. ที่คอม/iPad ที่ ER เปิดหน้า **`.../rak-er-jnh/scan.html`** → ล็อกอินด้วยบัญชี `erscan`
4. อนุญาตการใช้กล้อง → เปิดค้างไว้ตลอดเวร พนักงานเดินมาแสดงบาร์โค้ดให้กล้องสแกน ระบบจะลงเวรให้อัตโนมัติตามเวลา

> พนักงานเปิดบาร์โค้ดของตัวเองได้จากแท็บ "หน้าหลัก" ในมือถือ แล้วยื่นให้เครื่องที่ ER สแกน

## ⏰ กติกาในระบบ

**เวร** (แบ่งอัตโนมัติตามเวลาที่สแกน): เช้า 08:00–16:00 • บ่าย 16:00–00:00 • ดึก 00:00–08:00

**การรายงาน**
- รายงานเพื่อนได้ 1 ครั้ง/คน/เวร • แก้ไขรายละเอียดได้ • รายงานได้หลายคนต่อเวร
- **เพิ่ม**รายงานใหม่ได้เฉพาะภายใน 3 ชม. หลังลงเวร (แท็บ "รายงาน")
- แท็บ **"การรายงานของฉัน"** — ย้อนดูรายงานที่ตนเองเขียน (แยกรายเดือน) และ **แก้ไขรายละเอียด/ลบได้ภายใน 7 วัน** หลังบันทึก
- ผู้ถูกรายงานครบ **5 ครั้ง** → ขึ้นสถานะ "ต้องพบแอดมิน" จนกว่าแอดมินใหญ่จะกด **ยกเลิกแจ้งเตือน**
- **แอดมินใหญ่** แก้ไข/ลบรายงานและคำชื่นชมได้จากหน้า Dashboard → รายละเอียดรายบุคคล

**ความพึงพอใจเวร**
- ให้คะแนนด้วย **แถบเลื่อน (slider) 0–10** เลื่อนน้ำหนักคะแนนเอง
- กดเลือก **ปัจจัย** ที่ทำให้ **พอใจ / ไม่พอใจ** ได้หลายข้อ (บุคคล, จำนวนผู้ป่วย, ความยากของเคส, การส่งเวร, เพื่อนร่วมงาน, อากาศ) + เขียนข้อเสนอแนะเพิ่มได้
- Dashboard สรุปปัจจัยพอใจ/ไม่พอใจ เรียงตามจำนวน และออก PDF ได้

**การมาสาย**
- แอดมินใหญ่ **แก้เวลาเข้าเวร** ของคนที่มาสายได้ (ระบบคำนวณนาทีสายใหม่อัตโนมัติ) หรือลบบันทึกเข้าเวรได้ จาก Dashboard → รายละเอียดคนมาสาย

**การชื่นชม**
- ชื่นชมตนเองไม่ได้ • ชื่นชมคนในเวรได้ **1–2 คน/เวร** • ใส่เหตุผลหรือไม่ก็ได้
- ชื่นชมได้ไม่เกิน 3 ชม. หลังลงเวร

**การออก PDF** — กดปุ่ม "ออกรายงาน PDF" แล้วเลือก "บันทึกเป็น PDF" (Save as PDF) ในหน้าต่างพิมพ์

## 📌 ข้อควรทราบ (ข้อจำกัดทางเทคนิค)

- ระบบใช้ Firebase Authentication (Email/Password) โดยแปลง username → `username@erjnh.web` อัตโนมัติ ผู้ใช้ไม่ต้องมีอีเมลจริง
- การ "ลบผู้ใช้" จะลบเฉพาะข้อมูลโปรไฟล์ (ผู้นั้นจะเข้าใช้งานไม่ได้) — หากต้องลบบัญชีล็อกอินถาวร ให้ลบใน Firebase Console → Authentication ด้วย
- แทนที่จะลบ แนะนำให้ใช้ "ระงับการใช้งาน" ในหน้าแก้ไขผู้ใช้
- Firebase แผนฟรี (Spark) เพียงพอสำหรับทีมขนาด ER ทั่วไป
